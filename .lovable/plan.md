

# Fix: Missing `await` on Image Compression Causes Broken Images

## The Problem

Images ARE being generated by Google Gemini, but they're uploaded with `.undefined` file extensions because `imageResultToBuffer()` is an async function called without `await`. This means:

- `extension` resolves to `undefined` (hence `.undefined` filenames)
- `buffer` is also `undefined` (corrupt/empty files uploaded)
- The resulting URLs point to broken files

You can see this in the logs: `blog-images/articles/...-featured.undefined`

## The Fix

Add the missing `await` keyword in two places within `supabase/functions/bulk-generate-articles/index.ts`:

**Line 484** (inside `generateAIImage`):
```
// Before (broken):
const { buffer, extension } = imageResultToBuffer(result);

// After (fixed):
const { buffer, extension } = await imageResultToBuffer(result);
```

**Line 670** (inside `generateInfographic`):
```
// Before (broken):
const { buffer, extension } = imageResultToBuffer(result);

// After (fixed):
const { buffer, extension } = await imageResultToBuffer(result);
```

## Impact

- All newly generated articles will have properly compressed images with correct `.jpg` extensions
- Previously generated articles with `.undefined` URLs will need the backfill system to regenerate their images
- No other files need changes
